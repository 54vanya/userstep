# План внедрения функций StepEdit Lite

Составлен 2026-07-06 по результатам изучения мануала и декомпиляции
`StepEdit_Lite.exe` (детали — `stepedit_lite_findings.md` в памяти проекта).
Порядок фаз важен: фаза 2 (выделение) — фундамент для фазы 3.

## Фаза 1 — дешёвые шорткаты

- [x] Ctrl+Y — redo (в дополнение к Ctrl+Shift+Z; CLAUDE.md уже заявляет Ctrl+Y)
- [x] Ctrl+Tab / Ctrl+Shift+Tab — циклический перебор табов (работает в standalone-PWA; в браузерной вкладке перехвачен браузером)
- [x] Ctrl+колесо — зум поля (`setFieldZoom`, нативный non-passive wheel-listener в `ChartGrid`)
- [x] Ctrl+N — новый таб, Ctrl+O — импорт .ucs, Ctrl+W — закрыть таб (с confirm при isDirty)
- [x] Клавиатурная навигация: ↑/↓ — строка, PgUp/PgDn — страница, Home/End — начало/конец
- [x] Расширить Rush (play speed) до 0.2–4 с шагом 0.1 (у оригинала пресеты 0.25/0.5/1/1.5/2/3/4; минимум 0.2 — чтобы шаг 0.1 попадал ровно в 1.0)
- [x] Вынести файловые операции в `services/fileActions.ts` (меню + шорткаты без дублирования)

## Фаза 2 — выделение (фундамент)

- [x] `selection` в `editorStore`: `rows` (диапазон строк одного блока) | `block` (блок целиком)
- [x] Shift+клик / Shift+drag в `useEditor.ts`, Esc — сброс, обычный клик — сброс
- [x] Ctrl+A — выделить целиком блок под курсором (кросс-блочного выделения нет: у блоков разные split)
- [x] Рендер подсветки отдельным слоем в `ChartGrid` (`selection-overlay`)
- [x] Shift+клик по рельсе блока (`BlockRail`) — выделить весь блок
- Примечание: у оригинала два уровня выделения (ячейки/блоки) с разным поведением Delete (бит 0x800). Недокументированный Shift+Alt-режим (0x7000) не тащим.

## Фаза 3 — операции над выделением

- [x] Delete/Backspace — удалить ноты в диапазоне (кросс-блочные холды цепочками); Delete по block-выделению удаляет блок
- [x] Ctrl+C/X/V — внутренний клипборд (`services/selectionOps.ts`, ноты с относительными row); вставка в начало выделения или от строки плейхеда; block-клипборд вставляет копию блока
- [x] X — flip horizontal (single: `col → 4−col`, double: `col → 9−col`), Y — flip vertical (реверс строк, holds пересчитываются), M — mirror (X+Y)
- [x] Ctrl+Shift+V — вставка со сдвигом колонок (+1 колонка за нажатие, с заворотом)

## Фаза 4 — операции над блоками

- [x] `splitBlockAt(blockId, row)` — разрез на два блока; holds через границу получают `continues`/`continued` (`utils/blockOps.ts`, кнопка Split here в попапе блока; строка — из выделения или плейхеда)
- [x] `mergeWithNext(blockId)` — конкатенация нот со сдвигом row, свойства от первого; строки второго пересчитываются под split первого, парные цепочки склеиваются (кнопка Merge with next)
- [x] `deleteBelow(blockId, row)` — усечение rowCount (кнопка Delete below)
- [x] Adjust BeatSplit: при смене split в попапе строки нот пересчитываются `round(row × new/old)` — ноты остаются на долях; столкнувшиеся схлопываются

## Фаза 5 — ввод нот

- [x] Live-запись при playback: `KeyZ/KeyQ/KeyS/KeyE/KeyC` → колонки 0–4, `Numpad1/7/5/9/3` → 5–9 (по `e.code`); tap на ближайшую строку к `getCurrentMs`, существующая нота в ячейке замещается
- [x] Alt+drag — серия тапов вместо hold (с ghost-предпросмотром; коммит одним undo-снэпшотом)

## Фаза 6 — полировка

- [x] Метроном beat sound: тики на каждой доле бита (акцент на первой доле такта; 440/660 Гц — ниже хит-саундов), чекбокс Metronome в тулбаре
- [x] Статус-бар под гридом: время/блок/строка/такт/бит под мышью (запись в DOM напрямую, без ререндеров)
- [x] Resize блока перетаскиванием нижней границы (полоска на рельсе, ghost-линия с числом строк)
- [x] Drag&drop `.ucs`/`.piu.json`/аудио в окно + `file_handlers` в PWA-манифесте (`launchQueue` в main.tsx)
