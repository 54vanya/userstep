# Клавиатура и выделение

Справка по клавиатурному вводу PIU StepMaker и нюансам работы выделения.
Раскладка вдохновлена StepEdit Lite (см. `PLAN_STEPEDIT.md`).

`Ctrl` везде означает «Ctrl или Cmd» (на macOS работают обе).

## Воспроизведение

| Клавиша | Действие |
|---|---|
| `Space` | Play / pause |

Пробел срабатывает всегда, кроме фокуса в текстовом поле (text/number/textarea/
select/contenteditable). На слайдерах, чекбоксах и кнопках тулбара пробел
**не** активирует контрол, а уходит в play/pause — это инвариант тулбара.

Rush (скорость): слайдер 0.2×–4× с шагом 0.1 (минимум 0.2, а не 0.25 как в
оригинале — чтобы шаг попадал ровно в 1.0). На скоростях ниже ~0.5 высота тона
падает (Web Audio `playbackRate` без сохранения питча).

## Файлы и вкладки

| Клавиша | Действие |
|---|---|
| `Ctrl+S` | Сохранить `.piu.json` (с editorSettings: scale, rush, позиция) |
| `Ctrl+N` | Новая вкладка (пустой чарт) |
| `Ctrl+O` | Импорт `.ucs` |
| `Ctrl+W` | Закрыть активную вкладку (confirm при несохранённых изменениях) |
| `Ctrl+Tab` / `Ctrl+Shift+Tab` | Следующая / предыдущая вкладка |

**Нюанс браузера:** в обычной браузерной вкладке `Ctrl+N`, `Ctrl+W` и
`Ctrl+Tab` зарезервированы браузером и до приложения не долетают. Полноценно
они работают в установленном standalone-PWA. На macOS `Cmd+N/W` — те же
ограничения, но `Ctrl+Tab` работает и в браузере.

## Undo / redo

| Клавиша | Действие |
|---|---|
| `Ctrl+Z` | Undo (глубина 50) |
| `Ctrl+Y` или `Ctrl+Shift+Z` | Redo |

## Навигация по чарту

| Клавиша | Действие |
|---|---|
| `↑` / `↓` | На одну строку (шаг = высота строки блока под курсором, зависит от его split) |
| `PgUp` / `PgDn` | На страницу (высота вьюпорта) |
| `Home` / `End` | В начало / конец чарта |
| Колесо мыши | Скролл |
| `Ctrl+колесо` | Зум поля (Zoom, 50–300%) |

Нюансы:
- Позиция при `↑`/`↓` снэпится на ближайшую линию сетки, так что серия нажатий
  идёт ровно по строкам даже после ручного скролла «между линиями».
- Во время воспроизведения навигация с клавиатуры отключена (вертикальный
  скролл на playback заблокирован — конвейер ведёт аудио).
- Стрелки не перехватываются, когда фокус на слайдере/инпуте (у range-инпутов
  стрелки двигают сам слайдер).

## Выделение

Модель — как в StepEdit Lite, два уровня:

1. **Диапазон строк** (`rows`) — непрерывный интервал строк **одного блока**,
   во всю ширину (все колонки).
2. **Блок целиком** (`block`) — выделение на уровне блока.

| Действие | Результат |
|---|---|
| `Shift+клик` по сетке | Выделить строку; если выделение уже есть в этом блоке — расширить диапазон до кликнутой строки |
| `Shift+drag` по сетке | Выделить диапазон от якоря до текущей строки |
| `Ctrl+A` | Выделить целиком блок под курсором (плейхедом) |
| `Shift+клик` по рельсе блока (справа) | Выделить блок целиком (block-уровень) |
| `Esc` | Снять выделение |
| Обычный клик по сетке | Снимает выделение (и выполняет обычное действие — ставит/удаляет ноту) |

Нюансы:
- **Выделение не пересекает границы блоков.** У блоков разные `Split`, поэтому
  «строка» в разных блоках — разная длительность; диапазон всегда живёт внутри
  одного блока. При Shift+drag за границу блока диапазон зажимается в блок якоря.
  По этой же причине `Ctrl+A` выделяет текущий блок, а не весь чарт.
- Shift+клик снэпится на ближайшую линию **без** мёртвых зон (в отличие от
  постановки нот, где между линиями редкого блока есть мёртвая зона).
- Выделение — состояние редактора, а не чарта: не сохраняется, не попадает в
  undo-историю, сбрасывается при переключении вкладки.
- Уровень выделения важен для операций: Delete по диапазону строк удаляет ноты
  в диапазоне, Delete по выделенному блоку — сам блок.

## Операции над выделением

Работают только при активном выделении (иначе клавиши свободны).

| Клавиша | Действие |
|---|---|
| `Delete` / `Backspace` | rows: удалить ноты в диапазоне; block: удалить блок (единственный блок не удаляется — очищаются его ноты) |
| `Ctrl+C` | Копировать выделение во внутренний клипборд |
| `Ctrl+X` | Вырезать (копировать + удалить) |
| `Ctrl+V` | Вставить: в начало выделения, а без выделения — от строки под плейхедом |
| `Ctrl+Shift+V` | Вставить со сдвигом колонок: каждое нажатие сдвигает ещё на +1 колонку (с заворотом по ширине чарта) — для вариаций паттернов |
| `X` | Flip horizontal — зеркало по колонкам (single: `col → 4−col`, double: `col → 9−col`) |
| `Y` | Flip vertical — реверс строк внутри диапазона (холды пересчитываются) |
| `M` | Mirror — `X` + `Y` разом (поворот на 180°) |

Нюансы:
- **Удаление и холды**: нота, частично пересекающая диапазон, удаляется целиком;
  кросс-блочный холд удаляется всей цепочкой (как при клике по нему).
- **Копирование и холды**: холд, частично попавший в диапазон, обрезается по
  границам; выродившийся в одну строку — становится tap.
- **Вставка** кладёт ноты от целевой строки, замещая перекрытые существующие;
  хвост, не влезший в блок, обрезается. Вставленное сразу выделяется (удобно
  для повторной вставки со сдвигом).
- **Клипборд блока** (после `Shift+клик` по рельсе + `Ctrl+C`): `Ctrl+V`
  вставляет копию блока целиком после целевого блока.
- **Трансформации X/Y/M** не трогают кросс-блочные холды (их нельзя развернуть
  в пределах одного блока) и ноты, лишь частично пересекающие диапазон.
  Перевёрнутая нота, столкнувшаяся с нетронутой, отбрасывается.
- Клипборд внутренний (не системный) и живёт до перезагрузки страницы.

## Ввод нот

| Действие | Результат |
|---|---|
| Клик по сетке | Поставить tap (клик по существующей ноте — удалить) |
| Drag вниз | Растянуть hold (drag вверх — отмена) |
| `Alt+клик` / `Alt+drag` | Серия тапов: tap на каждой строке, через которую прошла мышь (колонка фиксируется стартовой) |

### Ввод нот с клавиатуры

Клавиши-колонки работают в любой момент; обе раскладки (UCS Lite и
StepMania) активны одновременно — их клавиши не пересекаются.

- **Во время playback (live-запись)**: клавиша кладёт tap в свою колонку на
  ближайшую к текущему моменту строку (квантование на линии текущего Split
  автоматическое); занятая ячейка замещается.
- **На паузе**: клавиша действует как клик по строке под курсором — пустая
  ячейка получает tap, занятая очищается. Если клавишу держать и нажимать
  `↓`/`↑` — от поставленной ноты растягивается холд в любую сторону (вниз или
  вверх), обратное движение укорачивает его, в том числе через границы блоков;
  отпускание клавиши завершает жест, весь жест = один шаг undo.

Раскладка **UCS Lite** (как в StepEdit Lite):

| Колонка | 1P (0–4) | 2P double (5–9) |
|---|---|---|
| ↙ | `Z` | `NumPad 1` |
| ↖ | `Q` | `NumPad 7` |
| ● | `S` | `NumPad 5` |
| ↗ | `E` | `NumPad 9` |
| ↘ | `C` | `NumPad 3` |

Раскладка **StepMania**: верхний ряд цифр `1 2 3 4 5 6 7 8 9 0` → колонки
0–9 слева направо (`1`–`5` — 1P, `6`–`0` — вторая половина double).

Нюансы:
- Раскладка UCS Lite физически повторяет крест панели; для 2P нужен именно
  **цифровой блок** (NumPad), не верхний ряд цифр — как в StepEdit Lite.
- Клавиши читаются по физическому коду (`e.code`) — раскладка ОС не важна.
- Существующая нота в ячейке замещается; каждое нажатие — отдельный шаг undo.
- Помогает Rush ниже 1.0: медленное воспроизведение — точнее попадание.

## Прочее

- **Метроном** (чекбокс Metronome в тулбаре): тики на каждой доле бита во время
  playback, первая доля такта акцентирована. Тон ниже хит-саундов, слои
  различимы на слух.
- **Статус-бар** под гридом: время / номер блока / строка / такт / бит под
  курсором мыши.
- **Resize блока**: перетащить нижнюю границу блока за рельсу справа (курсор
  row-resize) — ghost-линия показывает новое число строк.
- **Drag&drop** в окно: `.ucs` и `.piu.json` открываются новой вкладкой, аудио
  грузится в активную. В установленном PWA файлы `.ucs`/`.piu.json` можно
  открывать через «Открыть с помощью…» (file_handlers).
